# CIS Controls (CIS Critical Security Controls)

**CIS Controls** are a prioritized set of best practices to improve cybersecurity posture. This repo uses **CIS Controls v8** (18 controls; formerly SANS Top 20).

## The 18 controls (v8)

| # | Control |
|---|---------|
| 1 | Inventory and Control of Enterprise Assets |
| 2 | Inventory and Control of Software Assets |
| 3 | Data Protection |
| 4 | Secure Configuration of Enterprise Assets and Software |
| 5 | Account Management |
| 6 | Access Control Management |
| 7 | Continuous Vulnerability Management |
| 8 | Audit Log Management |
| 9 | Email and Web Browser Protections |
| 10 | Malware Defenses |
| 11 | Data Recovery |
| 12 | Network Infrastructure Management |
| 13 | Network Monitoring and Defense |
| 14 | Security Awareness and Skills Training |
| 15 | Service Provider Management |
| 16 | Application Software Security |
| 17 | Incident Response Management |
| 18 | Penetration Testing |

## In this repo

- **`fair_risk_register.ipynb`** — Each FAIR risk is mapped to one or more CIS controls (e.g. Ransomware → CIS 4, 8, 11). The notebook includes a CIS coverage heatmap by ALE.
- Mitigation recommendations reference which controls to apply.
